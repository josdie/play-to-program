Welcome to RUR-PLE!  In RUR-PLE, you will find a robot named Reeborg who is controlled by Python programs.  These Python programs contain instructions that tell Reeborg to perform actions and tests that help Reeborg decide what to do.  Reeborg is placed in an environment that is surrounded by walls and can contain other walls as well as objects called beepers, which Reeborg can pick up, hold on to, and put down.  To help test and develop your understanding of Python, you will be given an environment and a Python program, and your task will be to predict what Reeborg will do when running the given program.  The instructions for Reeborg's actions and tests are below.

Summary of instructions

Reeborg's actions:

move()		- moves Reeborg forward one step
turn_left()	- turns Reeborg 90° to the left
turn_right()	- turns Reeborg 90° to the right
grab_beeper()	- pick up one of the beepers Reeborg is standing on
drop_beeper()	- puts down one of the beepers that Reeborg is holding
turn_off()	- turns Reeborg off when he has completed a task

Tests:

front_is_clear()	- returns True if there is no wall directly in front of Reeborg
left_is_clear()		- returns True if there is no wall on Reeborg's left side
right_is_clear()	- returns True if there is no wall on Reeborg's right side
facing_north()		- returns True if Reeborg is facing upward
carries_beepers()	- returns True if Reeborg is holding any beepers
on_beeper()		- returns True if Reeborg is standing on any beepers

Entering Your Prediction

To get started, click on the Robot: Code and Learn tab above, and then click on the Submit/Next button on the toolbar. 

An environment and program will be loaded.  Your task is to predict the sequence of actions that Reeborg will perform by running the given program, and input this prediciton using the keyboard and mouse.  If you click outside of the robot area while your entering your prediction, you may need to click the robot area to regain control of the robot.

The controls for Reeborg's actions are:

  • Move - up arrow key on the keyboard
  • Turn Left - left arrow key on the keyboard
  • Turn Right - right arrow key on the keyboard
  • Grab Beeper - 'g' key on the keyboard or the Grab beeper button on the toolbar 
  • Drop Beeper - 'd' key on the keyboard or the Drop beeper button on the toolbar 
  • Print text to console (Python print statement) - 'p' key on the keyboard or the Print text to console button on the toolbar
  • Undo a previously entered action - 'u' key on the keyboard or the Undo robot action button on the toolbar
  • Reset and start the prediction over from the beginning - Reset world button on the toolbar 
  • Turn Off - 'o' key on the keyboard or the Turn robot off button on the toolbar  

*** Don't forget to turn Reeborg off at the end of your prediction!!! ***

Errors

Certain actions can cause errors which make Reeborg complain and turn off.  If you believe an action in the program will cause an error, do not enter any more actions after the one that causes the error.  If you input an action that causes an error and you did not mean to, make sure you Undo that action (you will see that action disappear from the console window on the bottom right).

Actions which cause errors:

  • Move - when a wall is in front of Reeborg
  • Grab Beeper - when Reeborg is not standing on any beepers
  • Drop Beeper - when Reeborg is not holding any beepers

Checking your answer

Once you have entered your prediction, click on the Submit/Next button on the toolbar. 

RUR-PLE will then run the robot program, and you can watch and see what it really does.  You can control the speed with which the program runs by using the Adjust robot speed slider on the toolbar  .  As the program runs, the line of code that is currently executing will be highlighted in the code window on the left.  When the program is done running, RUR-PLE will pop up a dialog telling you whether or not your prediction was correct.

Once the program has run, if there were errors in your prediction, you can run the program again and see where your prediction and the actions that were actually taken by the robot differ.  The controls for this are on the toolbar and behave like a visual debugger, which will be familiar if you have ever used one.  These controls are:

  • - Run the program
  • - Execute one statement and move to the next statement
  • - Pause/Unpause execution of the program
  •    - Stop running the program

As the program executes, the line of code that is currently executing will again be highlighted in the code window on the left.  As the robot actions execute, the robot executing your predicted actions will be blue  and trace a blue line through the environment.  The robot executing with correct actions will be the normal gray color  and trace a green line through the environment.  Also, the next robot action that will execute will be highlighted in the console window on the bottom right.  These actions will be preceded by a word which indicates which robot will execute them, and those words are:

  • CORRECT:  Your prediction of this action was correct and both robots will perform the action
  • INCORRECT:  Your prediction of this action was incorrect, and only your robot will perform it
  • CORRECTED:  You did not predict this action, and the robot performing the correct actions will

Note:  Incorrect actions involving beepers (grab and drop) will not be reflected in the environment.

Note:  If the program is paused, make sure you run it to completion or stop it before proceeding to the next problem.

When you are satisifed that you understand the differences between your prediction and the correct actions of the robot, you may proceed to the next problem by clicking on the Submit/Next button on the toolbar.
